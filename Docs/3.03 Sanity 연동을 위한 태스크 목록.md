
## Sanity 연동을 위한 태스크 목록

현재 **코드 레벨은 ~90% 준비**되어 있고, 주로 **외부 설정 작업**이 남아있습니다.

---

### 🔴 Step 1 — 외부 작업 (수동, 웹에서 진행)

| # | 태스크 | 설명 |
|---|--------|------|
| A | **Sanity.io 프로젝트 생성** | [sanity.io/manage](https://www.sanity.io/manage) → 새 프로젝트 생성 → `PROJECT_ID` 확보 |
| B | **API 토큰 생성** | 프로젝트 설정 → API → Tokens → "Editor" 권한 토큰 생성 → `SANITY_API_TOKEN` 확보 |
| C | **CORS Origin 설정** | 프로젝트 설정 → API → CORS Origins → `http://localhost:3000` 추가 |

```env
7
```

### 🟡 Step 3 — Sanity 스키마 정의 (Sanity Studio 또는 웹 대시보드)

이미 `src/types/sanity.ts`에 타입이 정의되어 있고, `queries.ts`에 GROQ 쿼리가 준비되어 있습니다. **Sanity 측에 스키마 4개**를 만들어야 합니다:

| 스키마 | 용도 | 필드 |
|--------|------|------|
| `post` | 뉴스/활동 | title, slug, category, excerpt, body(Portable Text), mainImage, publishedAt |
| `education` | 평화학교 기수 | title, slug, curriculum[], isRecruiting, startDate, endDate |
| `teamMember` | 임원진 | name, role, bio, photo, order |
| `timeline` | 연혁 | year, title, description |

> 방법 2가지: **(a)** Sanity Studio를 프로젝트에 내장 (embedded) **(b)** 별도 Sanity Studio 프로젝트로 관리

### 🟢 Step 4 — 코드 수정 (자동화 가능)

| # | 태스크 | 파일 | 내용 |
|---|--------|------|------|
| 1 | `next.config.ts` 이미지 도메인 추가 | `next.config.ts` | `cdn.sanity.io` remotePatterns 설정 |
| 2 | LatestNews → Sanity 데이터 연동 | `page.tsx` + `LatestNews.tsx` | MOCK_NEWS → `LATEST_POSTS_QUERY` GROQ 쿼리 교체 |
| 3 | 뉴스 목록/상세 페이지 구현 | `news/page.tsx`, `news/[slug]/page.tsx` | ISR + Portable Text 렌더링 (Phase 2-4) |
| 4 | 타임라인 페이지 연동 | `about/history/page.tsx` | `TIMELINE_QUERY` 연동 (Phase 2-3) |
| 5 | 임원진 페이지 연동 | `about/team/page.tsx` | `TEAM_MEMBERS_QUERY` 연동 (Phase 2-3) |
