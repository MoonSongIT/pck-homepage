## 로컬 테스트 단계별 가이드

### 1단계: 토스페이먼츠 테스트 키 발급

토스페이먼츠 개발자센터에서 테스트 키를 받아야 합니다.

1. https://developers.tosspayments.com 접속
2. 회원가입/로그인
3. **개발 정보 > API 키** 에서 **테스트 키** 복사
   - `클라이언트 키` (test_ck_...)
   - `시크릿 키` (test_sk_...)

### 2단계: 환경변수 설정

`.env` 파일에 테스트 키를 추가하세요.

DB는 설정되어 있습니다. `.env`에 토스 키를 추가해야 합니다.

```
# .env 파일에 아래 2줄 추가 (발급받은 테스트 키로 교체)
NEXT_PUBLIC_TOSS_CLIENT_KEY=test_ck_여기에_클라이언트키
TOSS_SECRET_KEY=test_sk_여기에_시크릿키
```

### 3단계: DB 마이그레이션

phone 필드를 추가했으므로 DB를 업데이트해야 합니다.

```bash
npx prisma db push
```

### 4단계: 개발 서버 실행

```bash
npm run dev
```

### 5단계: 테스트 순서

| 순서 | 동작 | URL |
|------|------|-----|
| 1 | 후원 페이지 접속 | `http://localhost:3000/donate` |
| 2 | 금액 선택 (프리셋 또는 직접입력) | - |
| 3 | 이름/이메일/전화번호 입력 | - |
| 4 | 개인정보 동의 체크 | - |
| 5 | "후원하기" 버튼 클릭 | - |
| 6 | 토스 결제창에서 **테스트 카드** 결제 | 토스 테스트 모드 |
| 7 | 성공 시 감사 페이지 확인 | `/donate/success` |

**토스 테스트 결제 시**: 카드번호 등은 아무 값이나 입력 가능 (테스트 모드)

토스 키를 발급받으셨나요? 받으셨으면 `.env`에 추가해 드리겠습니다.


## 토스페이먼츠 테스트 키 발급 절차

### 방법 1: 가입 없이 바로 테스트 (문서용 테스트 키)

토스페이먼츠는 **회원가입 없이도** 사용 가능한 문서용 테스트 키를 제공합니다.

1. [토스페이먼츠 개발자센터](https://developers.tosspayments.com/) 접속
2. 상단 메뉴에서 **샌드박스** 클릭
3. 코드 작성 없이 결제 플로우를 바로 체험 가능

> 단, 이 방법은 우리 프로젝트에 키를 넣어 테스트하기엔 부적합합니다.

---

### 방법 2: 이메일 가입으로 테스트 키 발급 (권장)

#### Step 1. 회원가입
1. [developers.tosspayments.com](https://developers.tosspayments.com/) 접속
2. **회원가입** 클릭
3. **이메일 주소 + 전화번호만** 입력 (사업자번호 불필요)
4. 이메일 인증 완료

#### Step 2. API 키 확인
1. 로그인 후 좌측 메뉴에서 **API 키** 클릭
2. **개발 연동 체험 상점**이 자동 생성됨
3. 아래 2개 키를 복사:

| 키 종류 | 형식 | 용도 |
|---------|------|------|
| **클라이언트 키** | `test_ck_...` | 브라우저에서 결제창 호출 (공개 가능) |
| **시크릿 키** | `test_sk_...` | 서버에서 결제 승인 API 호출 (비밀 유지) |

#### Step 3. .env에 키 입력

키를 복사했으면 알려주세요, `.env`에 추가해 드리겠습니다.

```
NEXT_PUBLIC_TOSS_CLIENT_KEY=test_ck_여기에붙여넣기
TOSS_SECRET_KEY=test_sk_여기에붙여넣기
```

---

### 테스트 결제 시 참고사항

- 테스트 환경에서는 **실제 카드번호를 입력해도 돈이 빠지지 않습니다**
- 별도의 테스트 카드번호는 없고, **본인 카드번호 그대로** 사용
- 에러 시뮬레이션: `TossPayments-Test-Code` 헤더로 에러 재현 가능
- 가상계좌(무통장입금) 모의 입금도 가능
