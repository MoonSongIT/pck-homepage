기술 스택을 계층별로 나눠서 설명할게요. 집 짓는 구조에 비유하면 이해하기 쉽습니다.

---

**언어 — TypeScript**

JavaScript에 타입 검사를 추가한 언어입니다. JavaScript는 변수에 어떤 값이든 넣을 수 있어서 자유롭지만, 그만큼 실수가 많이 납니다. TypeScript는 "이 변수는 숫자만 들어와야 해", "이 함수는 문자열을 반환해야 해"처럼 규칙을 미리 정해두고, 규칙을 어기면 코딩 중에 바로 빨간 줄로 알려줍니다. 런타임 에러를 코딩 시점에 잡아주는 안전망입니다. strict mode는 이 검사를 가장 엄격하게 적용하는 옵션입니다.

---

**프레임워크 — Next.js 15 (App Router, React 19)**

React는 화면을 컴포넌트 단위로 만드는 라이브러리입니다. 버튼 하나, 카드 하나, 헤더 하나를 각각 독립적인 부품으로 만들고 조립하는 방식입니다.

Next.js는 React 위에 얹히는 프레임워크입니다. React만으로는 SSG, ISR, SSR 같은 렌더링 전략을 쓸 수 없고, 라우팅(페이지 이동)도 직접 설정해야 합니다. Next.js가 이 모든 것을 제공합니다. App Router는 Next.js 13부터 도입된 새로운 폴더 구조 방식으로, 폴더 이름이 곧 URL 주소가 됩니다.

---

**스타일링 — Tailwind CSS v4**

CSS를 클래스 이름으로 직접 적용하는 방식입니다. 기존에는 별도 CSS 파일을 만들어서 `.button { color: blue; font-size: 14px; }` 처럼 작성했습니다. Tailwind는 HTML에 바로 `class="text-blue-500 text-sm"` 처럼 씁니다. CSS 파일을 따로 관리하지 않아도 되고, 클래스 이름만 봐도 어떤 스타일인지 바로 알 수 있습니다. v4는 CSS Variables 기반으로 전면 재설계된 최신 버전입니다.

---

**UI 라이브러리 — shadcn/ui + Radix UI**

Radix UI는 버튼, 드롭다운, 모달, 탭 같은 UI 컴포넌트를 접근성(키보드 조작, 스크린리더)까지 완벽하게 구현해둔 라이브러리입니다. 단, 스타일이 없습니다. 기능만 있는 뼈대입니다.

shadcn/ui는 Radix UI 위에 Tailwind로 스타일을 입혀놓은 컴포넌트 모음입니다. 특이한 점은 npm으로 설치하는 게 아니라 소스코드를 내 프로젝트에 직접 복사해서 가져옵니다. 그래서 마음대로 수정할 수 있습니다. 요즘 Next.js 프로젝트의 가장 인기 있는 UI 조합입니다.

---

**애니메이션 — Framer Motion 11**

React용 애니메이션 라이브러리입니다. 스크롤하면 카드가 아래서 올라오는 효과, 페이지 전환 시 슬라이드 효과, 숫자가 올라가는 카운터 애니메이션 같은 것들을 CSS 애니메이션보다 훨씬 쉽게 구현할 수 있습니다. PCK 사이트의 Impact Counter(회원 수, 활동 국가 수 등)나 Hero 섹션 진입 효과에 씁니다.

---

**다국어 — next-intl**

한국어와 영어를 지원하기 위한 라이브러리입니다. 번역 텍스트를 별도 파일로 관리하고, URL 앞에 `/en/about` 처럼 언어 코드를 붙여서 라우팅합니다. next-intl은 Next.js App Router와 궁합이 가장 좋은 다국어 라이브러리입니다.

---

**CMS — Sanity.io v3**

CMS는 Content Management System의 약자로, 콘텐츠 관리 시스템입니다. 개발자가 아닌 운영자(단체 담당자)가 뉴스·활동 소식·행사 일정을 직접 작성하고 수정할 수 있는 관리자 화면을 제공합니다. Sanity는 코드로 스키마(데이터 구조)를 정의하면 관리자 UI를 자동으로 만들어주고, API로 데이터를 꺼내 쓸 수 있습니다. 비영리 단체 무료 플랜이 있습니다.

---

**DB & ORM — Supabase + Prisma**

Supabase는 PostgreSQL 데이터베이스를 클라우드에서 쉽게 쓸 수 있게 해주는 서비스입니다. 회원 정보, 후원 내역, 커뮤니티 게시글처럼 구조화된 데이터를 저장합니다. Firebase의 오픈소스 대안으로 불립니다.

Prisma는 ORM입니다. ORM은 Object Relational Mapping의 약자인데, 쉽게 말하면 SQL 쿼리를 직접 쓰지 않고 JavaScript 코드로 DB를 다룰 수 있게 해주는 번역기입니다. `SELECT * FROM users WHERE id = 1` 대신 `prisma.user.findUnique({ where: { id: 1 } })` 처럼 씁니다. 타입스크립트와 궁합이 매우 좋습니다.

---

**인증 — NextAuth.js v5**

로그인·회원가입·세션 관리를 처리하는 라이브러리입니다. 이메일/비밀번호 로그인과 카카오 소셜 로그인을 모두 지원합니다. 직접 구현하면 보안 취약점이 생기기 쉬운 영역이라 검증된 라이브러리를 쓰는 것이 표준입니다. v5는 Next.js App Router에 맞게 완전히 재설계된 버전입니다.

---

**결제 — 토스페이먼츠 API**

국내 비영리 단체 후원 결제에 가장 많이 쓰이는 결제 서비스입니다. 카드 결제, 계좌이체, 카카오페이를 모두 지원하고, 정기 후원 자동결제도 처리할 수 있습니다. 테스트 환경과 실제 환경 키를 분리해서 개발 중에 실제 돈이 빠져나가지 않도록 관리합니다.

---

**캐시 & Rate Limit — Upstash Redis**

Redis는 데이터를 메모리에 저장하는 초고속 저장소입니다. Upstash는 Redis를 서버리스 방식으로 쓸 수 있게 해주는 클라우드 서비스입니다. PCK 사이트에서 두 가지 용도로 씁니다. Rate Limiting은 특정 IP가 로그인을 1분에 5번 이상 시도하면 차단하는 식으로 무차별 공격을 막는 것이고, Cache는 자주 조회되는 데이터를 DB 대신 Redis에서 빠르게 꺼내는 것입니다.

---

**이미지 최적화 — next/image + Cloudflare Images**

next/image는 Next.js 내장 이미지 컴포넌트입니다. 이미지를 자동으로 WebP 포맷으로 변환하고, 화면 크기에 맞는 사이즈로 리사이징하고, 뷰포트에 들어올 때만 로드하는 lazy loading을 자동 처리합니다. Cloudflare Images는 전 세계 CDN 서버에 이미지를 배포해서 한국 사용자는 한국 서버에서, 해외 사용자는 가까운 서버에서 빠르게 받을 수 있게 합니다.

---

**폼 검증 — React Hook Form + Zod**

React Hook Form은 폼(입력 양식) 상태를 효율적으로 관리하는 라이브러리입니다. 후원 폼, 교육 신청 폼, 회원가입 폼의 입력값 추적과 제출 처리를 담당합니다.

Zod는 데이터 유효성 검사 라이브러리입니다. "이메일 형식이 맞는가", "전화번호가 숫자 11자리인가", "후원 금액이 1000원 이상인가" 같은 규칙을 코드로 정의합니다. 클라이언트(브라우저)와 서버 양쪽에서 같은 규칙으로 검증할 수 있어서 TypeScript와 세트로 쓰는 표준 조합입니다.

---

**기타 — @vercel/og, Resend, next-themes**

@vercel/og는 카카오톡이나 SNS에 링크를 공유할 때 자동으로 표시되는 미리보기 이미지(OG 이미지)를 동적으로 생성합니다. 뉴스 제목과 이미지를 자동으로 넣어줍니다.

Resend는 이메일 발송 서비스입니다. 후원 완료 감사 메일, 교육 신청 확인 메일, 뉴스레터 발송에 씁니다. 개발자 친화적인 API로 요즘 가장 빠르게 성장하는 이메일 서비스입니다.

next-themes는 다크모드 전환을 처리하는 라이브러리입니다. 시스템 설정 자동 감지, 수동 전환, 새로고침 후에도 설정 유지를 한 번에 처리합니다.
