## GitHub 리포 연결 및 push — 단계별 설명

### 전체 흐름

```
GitHub에서 리포 생성 → 로컬에 연결 → 코드 push → Actions 자동 실행
```

---

### 1단계: GitHub에서 빈 리포지토리 생성

1. [github.com](https://github.com) 로그인
2. 우측 상단 **+** 버튼 → **New repository** 클릭
3. 설정:
   - Repository name: `pck-homepage` (소문자)
   - Description: "팍스크리스티코리아 공식 웹사이트"
   - **Public** 또는 **Private** 선택
   - ⚠️ **README, .gitignore, LICENSE는 체크하지 않음** (이미 로컬에 있으므로)
4. **Create repository** 클릭

### 2단계: 로컬 리포에 GitHub 주소 연결

GitHub이 생성 직후 보여주는 안내 화면에서 주소를 복사한 뒤:

```bash
git remote add origin https://github.com/MoonSongIT/pck-homepage.git
https://github.com/MoonSongIT/pck-homepage.git
```

이 명령어는 "앞으로 `origin`이라고 하면 이 GitHub 주소를 가리켜라"는 뜻입니다.

### 3단계: 코드 push

```bash
# main 브랜치 push
git checkout main
git push -u origin main

# develop 브랜치도 push
git checkout develop
git push -u origin develop
```

`-u`는 "이 브랜치가 origin의 같은 이름 브랜치를 추적해라"는 뜻입니다. 한 번만 하면 이후부터는 `git push`만으로 충분합니다.

### 4단계: GitHub Actions 자동 실행

push가 완료되면:

1. GitHub 리포 페이지 → **Actions** 탭 클릭
2. `ci.yml`이 자동 실행된 기록이 보임 (develop push 트리거)
3. 초록색 체크 ✅ = 성공, 빨간 X ❌ = 실패

```
실행 과정:
  ① GitHub 클라우드 서버(ubuntu)가 켜짐
  ② 코드 체크아웃
  ③ Node.js 22 설치 + npm ci
  ④ npm run lint → 통과?
  ⑤ npx tsc --noEmit → 통과?
  ⑥ npm run build → 통과?
  ⑦ 전부 통과하면 ✅ 표시
```

### 5단계: 이후 개발 흐름

```
feature 브랜치 생성 → 코드 작성 → push → PR 생성 → CI 자동 실행
                                              ↓
                                     ✅ 통과 시 머지 가능
                                     ❌ 실패 시 수정 필요
```
