먼저 "상태관리"가 뭔지부터 짚고 갈게요.

---

**상태(State)란?**

웹사이트에서 "지금 이 순간의 데이터"입니다. 예를 들어 "지금 로그인한 사람이 누구냐", "지금 한국어 모드냐 영어 모드냐", "지금 다크모드냐 라이트모드냐" 같은 것들이 전부 상태입니다.

문제는 이 상태를 여러 컴포넌트가 공유해야 할 때 생깁니다. 헤더도 로그인 상태를 알아야 하고, 사이드바도 알아야 하고, 게시판도 알아야 하는데, 각자 따로 관리하면 엉망이 됩니다. 그래서 "상태관리 도구"가 필요합니다.

---

**Zustand — 전역 상태 관리**

Zustand는 독일어로 "상태"라는 뜻입니다. 발음은 "주스탄트"입니다.

역할을 한 줄로 표현하면 **앱 전체가 공유하는 메모장**입니다.

PCK 사이트에서 Zustand가 관리하는 것들을 보면, 언어 설정은 한국어인지 영어인지를 저장하고 헤더·푸터·본문 전체가 이 값을 보고 언어를 바꿉니다. 다크모드는 밝은 화면인지 어두운 화면인지를 저장하고 모든 컴포넌트가 이 값을 보고 색상을 결정합니다. 인증 상태는 지금 누가 로그인해 있는지를 저장하고 커뮤니티·마이페이지·후원내역 등 여러 페이지가 이 값을 보고 접근을 허용하거나 막습니다.

Zustand 이전에는 Redux라는 도구가 표준이었는데, 설정 코드가 너무 복잡했습니다. Zustand는 같은 역할을 훨씬 적은 코드로 할 수 있어서 요즘 표준으로 자리잡았습니다.

---

**TanStack Query — 서버 상태 관리**

TanStack Query는 원래 이름이 React Query였다가 바뀐 것입니다. TanStack은 이 라이브러리를 만든 팀 이름입니다.

역할을 한 줄로 표현하면 **서버에서 가져온 데이터를 똑똑하게 관리하는 창고**입니다.

Zustand와 차이점이 여기서 중요한데, Zustand가 관리하는 상태는 브라우저 안에서만 존재하는 값들입니다. 언어 설정, 다크모드는 서버와 상관없이 브라우저가 스스로 들고 있는 정보입니다.

반면 TanStack Query가 관리하는 상태는 서버에서 가져와야 하는 데이터입니다. PCK 사이트에서는 뉴스 목록, 활동 게시글, 행사 일정, 회원 커뮤니티 글 같은 것들이 여기에 해당합니다.

이걸 그냥 직접 fetch로 가져와도 되는데, TanStack Query를 쓰는 이유가 있습니다. 같은 데이터를 여러 컴포넌트에서 요청할 때 중복 요청을 자동으로 막아줍니다. 한 번 가져온 데이터는 일정 시간 캐시에 보관해서 다시 요청하지 않습니다. 데이터가 오래됐다고 판단되면 백그라운드에서 자동으로 새로 가져옵니다. 로딩 중인지, 에러가 났는지, 성공했는지 상태를 자동으로 추적해줍니다.

---

**둘을 함께 쓰는 이유**

결국 관리하는 데이터의 성격이 다릅니다.

Zustand는 서버 없이 브라우저가 스스로 들고 있는 값, 즉 언어·다크모드·로그인 여부처럼 앱 전체가 공유해야 하는 설정값을 다룹니다.

TanStack Query는 서버에서 가져와야 하는 값, 즉 DB에 저장된 뉴스·게시글·행사 데이터처럼 네트워크 통신이 필요한 데이터를 다룹니다.

이 둘을 합치면 앱에서 발생하는 거의 모든 상태를 깔끔하게 처리할 수 있어서 Next.js 프로젝트의 현대적인 표준 조합으로 자리잡았습니다.
