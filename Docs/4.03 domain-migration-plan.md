# PCK 웹사이트 — 도메인 마이그레이션 계획

> 최종 수정: 2026-03-25
> 상황: 기존 도메인 `paxchristikorea.org`의 관리 권한 미확보 상태

---

## 1. 현황 및 전략

### 문제

- 기존 도메인 `paxchristikorea.org`는 이전 홈페이지에서 사용 중
- 도메인 관리자(레지스트라 계정 소유자)를 현재 알 수 없음
- 도메인 이전/DNS 변경이 즉시 불가능

### 전략: 2단계 배포

```
[1단계] 임시 도메인으로 배포 + 테스트
          ↓ (도메인 권한 확보 후)
[2단계] 원래 도메인으로 전환
```

## 2. 1단계 — 임시 도메인 배포

### 2-1. 임시 도메인 선택지

| 옵션 | 도메인 예시 | 비용 | 장점 | 단점 |
|------|------------|------|------|------|
| **A. Vercel 기본 도메인** | `pck-homepage.vercel.app` | 무료 | 즉시 사용, SSL 자동 | 비공식 URL |
| **B. 임시 도메인 구매** | `paxchristikorea.kr` | 연 ~1만원 | 공식적, 카카오/토스 연동 가능 | 비용 발생 |
| **C. 임시 도메인 구매** | `pck-korea.org` | 연 ~1.5만원 | 기존과 유사 | 비용 발생 |
| **D. 서브도메인 활용** | `new.paxchristikorea.org` | 무료 | 기존 도메인 활용 | ❌ DNS 권한 필요 |

### 2-2. 권장안

**개발/테스트 단계**: 옵션 A (Vercel 기본 도메인) — 무료, 즉시 사용 가능

**외부 연동 테스트 필요 시**: 옵션 B or C (임시 도메인 구매) — 아래 이유:
- 카카오 로그인 Redirect URI에 커스텀 도메인 필요
- 토스페이먼츠 실결제 시 도메인 등록 필요
- Google Search Console 등록 시 소유 도메인 필요

### 2-3. 임시 도메인 배포 절차

#### Step 1: Vercel 기본 도메인으로 배포

```
1. Vercel 프로젝트 생성 (deployment-guide.md §4 참조)
2. 환경변수 설정 — NEXTAUTH_URL을 임시 URL로 설정:
   NEXTAUTH_URL=https://pck-homepage.vercel.app
3. PR #13 머지 → 자동 배포
4. https://pck-homepage.vercel.app 접속 확인
```

#### Step 2: (선택) 임시 커스텀 도메인 연결

```
1. 도메인 구매 (가비아, Namecheap 등)
2. Vercel 대시보드 → Settings → Domains → 임시 도메인 추가
3. DNS 레코드 설정:
   A     @     76.76.21.21
   CNAME www   cname.vercel-dns.com
4. 환경변수 업데이트:
   NEXTAUTH_URL=https://paxchristikorea.kr  (예시)
5. 외부 서비스 콜백 URL 등록:
   - 카카오: https://paxchristikorea.kr/api/auth/callback/kakao
   - 토스: https://paxchristikorea.kr/donate/success
6. Vercel 재배포 (환경변수 변경 시 자동 재배포)
```

### 2-4. 임시 도메인 환경에서 변경되는 설정

| 설정 | 원래 값 | 임시 값 |
|------|---------|---------|
| `NEXTAUTH_URL` | `https://paxchristikorea.org` | `https://pck-homepage.vercel.app` 또는 임시 도메인 |
| 카카오 Redirect URI | `https://paxchristikorea.org/api/auth/callback/kakao` | 임시 도메인으로 변경 |
| 토스 성공/실패 URL | `https://paxchristikorea.org/donate/...` | 임시 도메인으로 변경 |
| sitemap.xml 내 URL | `https://paxchristikorea.org/...` | 코드 내 하드코딩 수정 필요 |
| robots.txt Sitemap | `https://paxchristikorea.org/sitemap.xml` | 코드 내 하드코딩 수정 필요 |
| JSON-LD url 필드 | `https://paxchristikorea.org` | 코드 내 하드코딩 수정 필요 |
| OG 이미지 URL | `https://paxchristikorea.org/api/og` | 자동 (상대경로 사용 시) |

### 2-5. 코드 내 하드코딩된 도메인 처리

사이트 URL을 환경변수로 통합 관리하여, 도메인 전환 시 코드 수정 없이 환경변수만 변경하면 됩니다.

#### 수정 대상 파일

```typescript
// src/lib/constants/site.ts (신규 생성)
export const SITE_URL = process.env.NEXT_PUBLIC_SITE_URL
  ?? process.env.NEXTAUTH_URL
  ?? 'https://paxchristikorea.org'
```

```
// 환경변수 추가
NEXT_PUBLIC_SITE_URL=https://pck-homepage.vercel.app  // 임시
```

#### 수정이 필요한 파일 목록

| 파일 | 현재 | 수정 후 |
|------|------|---------|
| `src/app/sitemap.ts` | `https://paxchristikorea.org` 하드코딩 | `SITE_URL` 사용 |
| `src/app/robots.ts` | `https://paxchristikorea.org` 하드코딩 | `SITE_URL` 사용 |
| `src/app/layout.tsx` | JSON-LD에 URL 하드코딩 | `SITE_URL` 사용 |
| 각 `page.tsx` | `metadataBase` 하드코딩 | `SITE_URL` 사용 |

## 3. 기존 도메인 권한 확보 방법

### 3-1. 도메인 관리자 추적 (WHOIS 조회 결과)

```
■ 이미 확인된 정보:
  Registrar:    Inames Co., Ltd. (도메인 구매처)
  Name Server:  ns.gabia.co.kr / ns.gabia.net / ns1.gabia.co.kr
  DNS 관리:     가비아 (gabia.com)

■ 추가 확인 필요:
  - Registrant (등록자) 이름/이메일 → WHOIS 비공개 설정일 수 있음
  - Expiration Date (만료일) → 만료 전 갱신 여부 확인
  - 가비아 계정 소유자 → 단체 내부 확인 필요

■ 단체 내부 확인 사항:
  - 이전 웹사이트 담당자에게 문의
  - 단체 이메일 관리자 (도메인 등록 시 사용한 이메일)
  - 재정 기록 (가비아/Inames 결제 내역으로 계정 특정)
  - 가비아 고객센터 (1544-4370) 에 단체 증빙서류로 소유권 증명 문의
```

### 3-2. 도메인 권한 이전 시나리오

| 시나리오 | 조건 | 절차 |
|----------|------|------|
| **A. 관리자 협조** | 관리자를 찾아 협조 받을 수 있는 경우 | 레지스트라에서 관리자 변경 또는 DNS 설정 변경 |
| **B. 레지스트라 직접 연락** | 단체 명의로 등록된 경우 | 단체 증빙서류로 레지스트라에 소유권 증명 → 계정 복구 |
| **C. 도메인 만료 대기** | 갱신되지 않을 경우 | 만료 후 재등록 (보통 만료 후 30~90일 대기) |
| **D. 분쟁 해결** | 소유권 다툼 시 | UDRP(통일도메인분쟁해결정책) 또는 KRDRP(한국) 신청 |

### 3-3. WHOIS 조회 명령어

```bash
# 터미널에서 직접 조회
whois paxchristikorea.org

# 또는 온라인 도구
# https://whois.domaintools.com/paxchristikorea.org
# https://www.whois.com/whois/paxchristikorea.org
```

## 4. 2단계 — 원래 도메인으로 전환

### 4-1. 전환 전 준비 (도메인 권한 확보 후)

```
1. DNS 관리 패널 접근 확인
2. 기존 DNS 레코드 백업 (스크린샷)
3. 기존 이메일 서비스 여부 확인 (MX 레코드)
   → 이메일을 사용 중이면 MX 레코드를 건드리지 않도록 주의
4. 전환 시간대 결정 (트래픽이 적은 시간, 예: 화요일 새벽 2시)
```

### 4-2. DNS 전환 절차

```
Step 1: Vercel에 도메인 추가
  → Vercel 대시보드 → Settings → Domains → paxchristikorea.org 추가

Step 2: 가비아 DNS 레코드 변경
  접속: gabia.com 로그인 → My 가비아 → 서비스 관리 → DNS 관리
       → paxchristikorea.org [설정] → DNS 레코드 관리

  기존 A/AAAA 레코드 삭제 (웹 관련만)
  신규 추가:
    타입    호스트    값                         TTL
    A       @        76.76.21.21                300
    CNAME   www      cname.vercel-dns.com       300

  ⚠️ MX 레코드는 절대 삭제하지 마세요 (이메일 서비스 유지)
  ⚠️ TXT 레코드 (SPF, DKIM — Resend 용)도 유지
  ⚠️ 가비아 호스트명에 도메인을 붙이지 말고 앞부분만 입력

Step 3: SSL 인증서 발급 대기
  → Vercel이 Let's Encrypt 인증서를 자동 발급 (보통 수 분)

Step 4: DNS 전파 확인
  → nslookup paxchristikorea.org
  → 76.76.21.21 응답 확인 (전파에 최대 48시간, 보통 10분)
```

### 4-3. 환경변수 업데이트

```bash
# Vercel 대시보드에서 변경
NEXTAUTH_URL=https://paxchristikorea.org
NEXT_PUBLIC_SITE_URL=https://paxchristikorea.org
```

### 4-4. 외부 서비스 콜백 URL 업데이트

| 서비스 | 설정 위치 | 변경 내용 |
|--------|-----------|-----------|
| 카카오 로그인 | developers.kakao.com → 앱 → 카카오 로그인 | Redirect URI: `https://paxchristikorea.org/api/auth/callback/kakao` |
| 토스페이먼츠 | developers.tosspayments.com | 허용 도메인에 `paxchristikorea.org` 추가 |
| Resend | resend.com → Domains | `paxchristikorea.org` 도메인 인증 |
| Google Search Console | search.google.com/search-console | 새 속성 추가 + sitemap 제출 |

### 4-5. 임시 도메인 정리

```
1. Vercel에서 임시 도메인 제거
   → Settings → Domains → 임시 도메인 삭제

2. (임시 도메인을 구매한 경우)
   → 리다이렉트 설정: 임시 도메인 → paxchristikorea.org (301)
   → 또는 갱신 중지하여 자연 만료

3. 카카오/토스에서 임시 도메인 콜백 URL 제거

4. Google Search Console에서 임시 도메인 속성 삭제
```

### 4-6. 전환 후 검증 체크리스트

- [ ] `https://paxchristikorea.org` 접속 — 새 사이트 표시
- [ ] `https://www.paxchristikorea.org` → `https://paxchristikorea.org` 리다이렉트
- [ ] `http://paxchristikorea.org` → `https://` 리다이렉트 (HTTPS 강제)
- [ ] SSL 인증서 유효 (자물쇠 아이콘)
- [ ] `/sitemap.xml` — 올바른 URL 표시
- [ ] `/robots.txt` — 올바른 Sitemap URL
- [ ] 카카오 로그인 정상 동작
- [ ] 토스 결제 정상 동작
- [ ] 이메일 발송 정상 (Resend)
- [ ] Google Search Console에서 sitemap 제출
- [ ] 기존 도메인으로의 외부 링크(SNS 등) 정상 연결 확인

                    │

## 6. 위험 요소 및 대응

| 위험 | 영향 | 대응 |
|------|------|------|
| 도메인 관리자를 끝내 찾지 못함 | 기존 도메인 사용 불가 | 임시 도메인을 정식 도메인으로 전환 (`.kr` 권장) |
| 도메인 만료 후 제3자가 선점 | 기존 도메인 영구 상실 | 만료 전 갱신 시도, 또는 백오더 서비스 이용 |
| DNS 전환 중 다운타임 | 접속 불가 (최대 48시간) | TTL을 300초로 낮춰 전파 속도 향상, 심야 작업 |
| 이메일 MX 레코드 삭제 실수 | 단체 이메일 수신 불가 | DNS 변경 전 기존 레코드 전체 백업 필수 |
| 기존 사이트 SEO 점수 손실 | 검색 순위 하락 | 301 리다이렉트 + Google Search Console 재등록 |
